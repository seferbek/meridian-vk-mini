# VK Mini Apps запуск

1. Создайте мини-приложение в кабинете разработчика VK.
2. Скопируйте `app_id` приложения.
3. В корне проекта замените `app_id` в `vk-hosting-config.json` и `vk-tunnel-config.json`.
4. Для локальной отладки в VK:
   - запустите `npm run dev:vk`
   - локальный VK-сервер будет доступен на `http://localhost:3001/`
   - в отдельном терминале запустите `npm run tunnel`
   - в настройках приложения в кабинете VK включите режим разработки и укажите URL, который выдаст tunnel
5. Для публикации на хостинг VK:
   - для быстрой отладки запустите `npm run deploy`
   - это обновит только `dev`-URL и обычно работает заметно быстрее
   - для production используйте `npm run deploy:prod`
   - если нужно обновить и `dev`, и `prod`, используйте `npm run deploy:all`
   - подтвердите авторизацию, если CLI попросит
   - после успешного деплоя укажите выданный URL как основной адрес приложения

Backend-авторизация:

1. На сервере задайте переменные окружения:
   - `VK_APP_SECRET` — защищённый ключ мини-приложения из кабинета VK
   - `VK_APP_ID` — id мини-приложения VK, чтобы backend дополнительно сверял `vk_app_id`
   - `VK_LAUNCH_PARAMS_MAX_AGE_SECONDS` — опционально, максимальный возраст `vk_ts`; можно не задавать, если нужны долгоживущие web-сессии VK
   - `CORS_ORIGIN` — домен фронта или `*` для быстрой отладки
   - `PORT` — порт backend, по умолчанию `8787`
2. Запустите backend:
   - `npm run backend`
3. На фронте задайте `VITE_API_BASE_URL` с публичным URL backend.
4. Соберите и задеплойте фронт обычным способом.

Как это работает:

- Внутри VK Mini Apps фронт отправляет подписанные launch params на backend.
- Backend проверяет `sign` через `VK_APP_SECRET` при каждом защищённом обращении: `/api/auth/vk`, `/api/session`, `/api/state`.
- Backend доверяет `vk_user_id` только после успешной проверки подписи и автоматически создаёт/находит профиль пользователя.
- Отдельной регистрации, логина по email/password и локальных backend-сессий больше нет.

Важно:

- Для VK-модерации `VITE_API_BASE_URL` и `VK_APP_SECRET` обязательны. Если launch params есть, но backend не настроен или подпись не проходит, приложение показывает ошибку.
- В браузере вне VK приложение может использовать локальное состояние только для разработки. В production открывайте его через VK Mini Apps.
- Для production backend должен быть размещён на постоянном хостинге с сохранением `server/data/db.json` или с настоящей БД.
